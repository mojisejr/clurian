# Clurian Feature Documentation (Thai)

เอกสารนี้รวบรวมรายการฟีเจอร์และการทำงานทั้งหมดของระบบ Clurian - Orchard Manager โดยวิเคราะห์จากต้นแบบ `docs/index.html`

## 1. **Dashboard & Overview (หน้าภาพรวม)**
เป็นหน้าแรกที่ผู้ใช้งานจะเห็นเมื่อเข้าสู่ระบบ เพื่อดูสถานะภาพรวมของสวน
*   **แสดงสถิติสำคัญ**:
    *   จำนวนต้นไม้ทั้งหมดในสวน (Total Trees)
    *   จำนวนต้นไม้ที่ต้องดูแลพิเศษ/ป่วย (Sick Trees)
*   **ค้นหาและกรองข้อมูล (Filter & Search)**:
    *   ค้นหาต้นไม้ด้วยรหัส (Code) หรือสายพันธุ์
    *   กรองดูเฉพาะโซนที่ต้องการ (Zone Filter) เช่น "โซน A", "ริมน้ำ" หรือ "ทุกโซน"
*   **รายการต้นไม้ (Tree List)**:
    *   แสดงรายการแบบการ์ด โดยเรียงตามรหัสต้น (A01, A02...)
    *   แสดงสถานะด้วยสี (ปกติ=เขียว, ป่วย=ส้ม, ตาย=แดง)
    *   หน้ารายการมีการแบ่งหน้า (Pagination)

## 2. **Authentication (ระบบยืนยันตัวตน)**
*   **Line Login**: เข้าสู่ระบบผ่าน Line Account เพื่อความสะดวก
*   **Multi-Orchard Support**:
    *   ผู้ใช้หนึ่งคนสามารถเป็นเจ้าของหรือดูแลได้หลายสวน
    *   มีเมนู Dropdown ให้สลับเปลี่ยนสวนที่ต้องการจัดการได้ทันที
    *   สามารถสร้างสวนใหม่ (Create Orchard) เพิ่มได้

## 3. **Tree Management (ระบบจัดการทะเบียนต้นไม้)**
บริหารจัดการข้อมูล Master Data ของต้นไม้แต่ละต้น
*   **ลงทะเบียนต้นใหม่ (Register New Tree)**:
    *   ระบุรหัสต้น (Code), โซน (Zone) *รองรับการเพิ่มโซนใหม่*, ชนิดพืช, สายพันธุ์, และวันที่ปลูก
*   **แก้ไขสถานะต้นไม้ (Update Status)**:
    *   เปลี่ยนสถานะสุขภาพได้ 3 ระดับ: `Healthy` (ปกติ), `Sick` (ป่วย), `Dead` (ตาย)
    *   การเปลี่ยนสถานะจะถูกบันทึกลง Log โดยอัตโนมัติ
*   **ระบบปลูกซ่อม (Replanting)**:
    *   สำหรับต้นที่ "ตาย" จะมีปุ่ม "ปลูกซ่อม"
    *   ระบบจะย้ายต้นเดิมไปเก็บใน Archive (เปลี่ยนรหัสเป็น `_HIST_`)
    *   สร้างต้นใหม่ในรหัสเดิมและโซนเดิมให้ทันที

## 4. **Activity Logging (ระบบบันทึกกิจกรรม)**
หัวใจสำคัญของการดูแลสวน รองรับการทำงานจริงของเกษตรกร
*   **Individual Log (บันทึกรายต้น)**:
    *   บันทึกกิจกรรมเจาะจงเฉพาะต้นนั้นๆ เช่น "รักษาโรครากเน่า", "ขูดแผลทายา"
    *   เข้าถึงได้ผ่านหน้า Tree Detail ของต้นนั้นๆ
*   **Batch Log (บันทึกเหมาแปลง)**:
    *   บันทึกกิจกรรมที่ทำเหมือนกันทั้งโซนหรือทั้งสวน เช่น "ใส่ปุ๋ย", "พ่นยา", "ตัดหญ้า"
    *   สามารถเลือก Target Zone ได้

## 5. **Health Monitoring & Follow-up (ระบบติดตามอาการ)**
ช่วยไม่ให้ลืมต้นไม้ที่ป่วยหรือต้องดูอาการต่อเนื่อง
*   **Follow-up Date**: เมื่อบันทึกการรักษา สามารถระบุ "วันนัดดูอาการ" ได้
*   **Follow-up Dashboard**: ในหน้าประวัติ (History) มี Tab แยกสำหรับรายการ "ติดตาม/นัดหมาย" โดยเฉพาะ
*   **Countdown**: แสดงจำนวนวันคงเหลือ (อีก 3 วัน, วันนี้, เกินกำหนด) พร้อมสีแจ้งเตือน
*   **Tracking Updates**:
    *   เมื่อถึงวันนัด สามารถกดอัพเดทอาการได้ว่า "หายดีแล้ว" (Cured) หรือ "ยังไม่หาย" (Continue)
    *   ถ้าหายแล้ว ระบบจะเสนอให้ปรับสถานะต้นไม้กลับเป็น "ปกติ"

## 6. **History & Reporting (ประวัติและรายงาน)**
*   **Tree History**: ดูประวัติการดูแลย้อนหลังทั้งหมดของแต่ละต้น (รวมทั้ง Log รายต้น และ Log เหมาแปลงที่ครอบคลุมต้นนั้น)
*   **QR Code**:
    *   สร้าง QR Code ประจำต้นไม้ สำหรับนำไปติดที่หน้าต้น
    *   **Print Mode**: จัดหน้า A4 สำหรับสั่งพิมพ์ QR Code จำนวนมากพร้อมกันได้ทันที

## 7. **Utility Features**
*   **Mobile-First Design**: ออกแบบมาให้ใช้งานง่ายบนมือถือ
*   **Offline Experience** (Future): ออกแบบ UI ให้รองรับการทำงานในพื้นที่สัญญาณน้อย (แม้ปัจจุบันจะเป็น Web App)
